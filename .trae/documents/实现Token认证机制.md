# 实现Token认证机制

## 问题分析
- 当前服务器API没有认证机制，任何请求都可以访问
- CLI工具直接发送请求，未携带任何认证信息
- 配置文件中已有Token字段，但未被使用

## 解决方案

### 1. 服务器端实现
- 添加认证中间件，验证请求头中的Authorization token
- 从配置文件读取允许的token
- 对所有API端点应用认证

### 2. CLI工具实现
- 修改HTTP客户端，从配置文件读取token
- 在所有请求中添加Authorization头
- 确保配置向导支持token设置

### 3. 配置管理
- 确保配置文件中的MCP.Token字段被正确使用
- 提供清晰的错误信息，当token无效时

## 实现步骤
1. 在EyuBot.App中添加认证中间件
2. 修改API端点，添加认证检查
3. 更新EyuBot.CLI，从配置读取token并添加到请求头
4. 测试认证机制是否正常工作

## 预期效果
- 未携带有效token的请求将被拒绝
- CLI工具能够正常访问API（携带正确token）
- 安全性得到提升，防止未授权访问
